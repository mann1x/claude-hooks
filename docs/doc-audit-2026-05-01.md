# Documentation audit — 2026-05-01

Working artifact for the doc-refresh pass kicked off on 2026-05-01.
Captured by a detective-agent inventory of code vs docs, then turned
into a phased fix plan. The audit itself is read-only — fixes land in
follow-up commits.

## 1. Executive summary

Three major feature areas shipped in the last ~2 weeks (LSP engine
Phases 0–4, the proxy `sp_effort` dashboard view + daily health
timer, the caliber-proxy native-tools agent) that the README either
understates or omits entirely. The headline gap (LSP engine) is real
but partially documented (lines 211–223 plus a 483-line runbook); the
bigger uncovered surface is everything else: the
`claude-hooks-health.timer`, in-stream stop-phrase scanner
(`scan_stop_phrases`), `pgvector_mcp` system-wide MCP server,
`caliber_proxy` (config knob committed but unmentioned), and the
entire `PostToolUse` toml-advisor + dashboard service. The README
contains stale numeric claims ("v0.4.0", "524 passed"; actual
collected = 1527) and `CLAUDE.md` is materially behind ("58 tests",
PreToolUse-only hook table, no PostToolUse, no LSP, no proxy stack).
Skill files in `.agents/` / `.cursor/` are git-modified and lag
behind the `.claude/` copy. The README's section ordering (proxy at
v0.5+ → code-graph at v0.6+ → IDE-loop at v0.7+) tracks feature
history but contradicts the project version pinned at `0.4.0` in
`pyproject.toml`.

## 3. Severity A — Missing from README

1. **Daily proxy health timer** — `claude-hooks-health.{service,timer}` + `scripts/proxy_health_oneliner.py`. Added 2026-05-01. Single-line stop-phrase trend summary; systemd `Persistent=true`, fires at 09:07 UTC. Zero references in README.
2. **In-stream stop-phrase scanner** (`proxy.scan_stop_phrases`). Backed by `claude_hooks/proxy/stop_phrase_guard.py`, the YAML catalog at `config/stop_phrases.yaml`, the `sp_*` columns in `proxy/stats_db.py`, and the `sp_effort` view shipped in commit `d47ef0a`. AGENTS.md describes it; README never names it once.
3. **`pgvector_mcp` system-wide stdio MCP server**. Entry point in `pyproject.toml` `[project.scripts]`, bin shims, systemd unit. Lets Cursor / Codex / OpenWebUI use the same Postgres store. Mentioned only inside one row of the providers table.
4. **`caliber_proxy` config block + native-tools agent loop**. `config/claude-hooks.example.json` ships a fully-baked block (port 38090, env-var knobs). README mentions caliber-proxy but never references the JSON config key, the 38090 port, or the `survey_project` tool.
5. **PostToolUse `toml_comment_advisor`**. Default-on, scoped to LSP-engine TOML edits. Zero docs.
6. **Proxy SSE thinking-depth metrics (S3, S4)**. `proxy/sse.py` parses extended-thinking signature bytes, delta count, output-token splits, visible/redacted thinking, per-tool counts. AGENTS.md describes; README's Proxy section omits.
7. **`setup-compile-aware` slash command**. Skill ships, README's slash-command table omits it.
